# 项目 SEO 优化指南

**文件名：** `seo.md`
**目标：** 为开发者和 AI（Cursor）提供清晰、可操作的 SEO 实施标准，确保项目代码符合最佳搜索引擎优化实践。

### 1. 核心元标签规范

### Title 页面标题
- 为每个页面提供独特的标题，简洁准确地描述页面内容
- 标题长度保持在50-60字符以内（避免在搜索引擎结果中被截断）
- 将重要的关键词放在首位，但要自然流畅
- 在全站所有页面的标题结尾加上网站名称和域名
- 如果网站名称就是域名，则只需要加域名

### Description 页面描述
- 为每个页面提供独特的元描述，清晰反映页面传达的价值
- 字符数控制在150-160字符（包括空格）
- 包含最重要的关键词，在搜索结果中突出显示
- 避免关键词堆砌，保持自然流畅
- 可选择性使用引人注目的号召行动语句
- 体现独特的价值主张，如"学习"、"购买"等行动提示

### 2. 文本内容规范

### 内容质量标准
- 每个页面至少1200个单词以上
- 核心关键词密度控制在3%到5%
- 首要保证内容自然流畅，避免刻意堆砌关键词
- 内容要让用户觉得是专门为他编写的
- 提供有价值、实用的信息内容

### 3. 标题标签优化

### Heading 标题标签（H1到H6）
- 确保标题与描述的文本段落相关
- 标题要反映所覆盖文本的情感基调
- 避免使用简单的顺序标题如"第一章...第二章..."
- 不要过度使用标签和其中的关键词
- 保持对用户的可读性和友好性
- H1和Title可以相同，也可以根据需要进行调整

### 4. 语义化HTML结构

### 语义化标签使用优先级
使用语义化标签：`<header>`, `<nav>`, `<main>`, `<article>`, `<section>`, `<aside>`, `<footer>`

### 标题层级规范
- 每个页面必须有且仅有一个 `<h1>`
- 层级顺序：`h1 → h2 → h3 → h4` （禁止跳级）
- 标题应准确描述内容区块

### 5. 图片优化标准

### Alt 属性规范
- **内容图片：** 准确描述图片内容
- **文本图片：** alt文本包含图片中所有文字
- **装饰图片：** 使用 `alt=""`
- **禁止：** 关键词堆砌

### 6. 内部链接建设

### 锚文本优化原则
- **描述性：** 清晰说明目标页面内容
- **自然性：** 融入上下文，避免生硬
- **相关性：** 与目标页面主题匹配

### 7. Open Graph 社交分享

### 必需的基础标签
- og:title - 页面标题
- og:description - 页面描述
- og:image - 分享图片URL
- og:url - 页面完整URL
- og:type - website|article|product等

### 8. 结构化数据 (Schema.org)

### 常见类型
- Article - 文章页面
- Product - 产品页面
- Organization - 组织信息
- LocalBusiness - 本地商家

### 9. 性能与核心Web指标

### 图片优化策略
- **格式优先：** WebP → JPEG/PNG
- **尺寸适配：** 提供多尺寸srcset
- **懒加载：** 非首屏图片使用loading="lazy"

### 代码分割与懒加载
- 动态导入重型组件
- 路由级别代码分割
- 组件级别懒加载

### 10. 移动端友好性

### Viewport 配置
必须配置：`<meta name="viewport" content="width=device-width, initial-scale=1.0">`

### 触摸目标尺寸
- 最小点击目标：44×44像素
- 足够的间距防止误触

### 11. 可访问性 (A11y) 基础

### 键盘导航支持
- 所有交互元素可通过键盘访问
- 可见的焦点指示器

### ARIA 标签使用
- 为交互元素提供aria-label
- 为导航区域提供aria-label
- 为表单元素提供aria-describedby

### 给 Cursor AI 的指令

当处理以下任务时，请参考本SEO指南：
1. **创建新页面** → 确保包含完整的meta标签和语义化结构
2. **优化现有页面** → 检查并修复SEO相关问题
3. **添加图片** → 必须包含合适的alt文本
4. **编写内部链接** → 使用描述性锚文本
5. **设置分享预览** → 实现Open Graph协议
6. **重要内容页面** → 考虑添加结构化数据
7. **编写文本内容** → 确保内容长度和关键词密度符合标准
8. **设置标题标签** → 优化H1-H6标签的层次结构

### 主动检查项
- [ ] 页面标题是否唯一且符合长度要求
- [ ] Meta描述是否完整且吸引人
- [ ] 文本内容是否达到1200单词以上
- [ ] 关键词密度是否在3%-5%范围内
- [ ] 图片是否都有合适的alt属性
- [ ] 标题层级是否正确
- [ ] 链接锚文本是否描述性
- [ ] 是否考虑移动端体验
- [ ] H1标签是否与页面内容高度相关
- [ ] 内容是否自然流畅，避免关键词堆砌
